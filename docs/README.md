# 42lib-flutter 문서

이 디렉토리는 프로젝트의 모든 문서를 체계적으로 관리합니다.

## 디렉토리 구조

### `/docs/architecture/`
시스템 설계 및 기술 아키텍처 문서
- 시스템 구조도
- 기술 스택 선정 이유
- 아키텍처 의사결정 기록 (ADR)

### `/docs/api/`
API 문서 및 계약(Contract) 명세
- API 엔드포인트 문서
- 요청/응답 스키마
- API 버전 관리 정책

### `/docs/guides/`
사용자 가이드 및 튜토리얼
- 설치 가이드
- 사용법 튜토리얼
- 모범 사례

### `/docs/processes/`
개발 프로세스 및 워크플로우
- 개발 프로세스 문서
- 코드 리뷰 가이드라인
- 릴리스 프로세스

### `/docs/decisions/`
아키텍처 의사결정 기록 (Architecture Decision Records)
- 중요한 기술적 결정 사항
- 의사결정 배경 및 근거
- 대안 검토 내용

## GitHub Wiki 연동

이 문서들은 GitHub Wiki와 동기화되어 검색 및 접근성을 향상시킵니다.

## 문서 작성 규칙

1. **언어**: 모든 문서는 한글로 작성
2. **형식**: Markdown 형식 사용
3. **커밋**: 문서 변경 시 관련 Issue 번호 포함
4. **리뷰**: Pull Request를 통한 문서 검토

## 기여 방법

1. 관련 GitHub Issue 생성
2. 해당 브랜치에서 문서 작성/수정
3. Pull Request 생성 (한글 설명 포함)
4. 리뷰 후 병합
