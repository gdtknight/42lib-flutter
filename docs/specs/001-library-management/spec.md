# 기능 명세서: 42 학습 공간 도서 관리 시스템

**기능 브랜치**: `001-library-management`  
**작성일**: 2025-12-17  
**상태**: 초안  
**입력**: 사용자 설명: "42 학습 공간에 비치된 학습용 도서 관리용 앱을 제작할 예정. 일반 사용자용 앱을 제작하고, 도서 관리자를 위한 기능들은 웹으로 제공할 예정. 도서 개수는 약 500개 정도이고 공간상 제약으로 최대 1000개 정도까지 늘어날 것으로 예상함. 우선적으로 사용자 앱을 통해 도서 정보 열람이 가능하도록 할 것이고, 42 API 연동을 통해 도서 대출까지 확장할 계획을 가지고 있음. 도서 관리자용 기능은 도서 추가, 제거, 조회, 대출 관련 정보 조회가 필수로 들어갈 예정이고, 희망 도서 목록 수집과 관련된 기능도 추가적으로 들어갈 예정. 희망 도서는 반기 내지 분기 1회 정도로 요청받을 계획."

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 스토리 1 - 모바일 앱에서 도서 탐색 및 검색 (우선순위: P1)

42 학습 공간의 학생들은 도서관 컬렉션에서 이용 가능한 도서를 찾아야 합니다. 모바일 앱을 열어 약 500-1000권의 도서 카탈로그를 탐색하고, 제목, 저자 또는 주제별로 검색하며, 대출 가능 여부를 포함한 각 도서의 상세 정보를 확인합니다.

**이 우선순위인 이유**: 이것은 핵심 가치 제안으로, 학생들이 학습 자료를 찾을 수 있게 합니다. 이 기능이 없으면 앱은 존재 의미가 없습니다. 이 스토리는 즉각적인 가치를 제공하며 단독으로 실행 가능한 MVP로 작동할 수 있습니다.

**독립적 테스트**: 앱을 실행하고, 도서 목록을 탐색하며, 검색을 수행하고, 도서 상세 정보를 확인함으로써 완전히 테스트할 수 있습니다. 학생들이 물리적 도서관을 방문하지 않고도 이용 가능한 학습 자료를 발견할 수 있도록 하여 가치를 제공합니다.

**디자인/UX 요구사항**:
- 시각적 정체성: 42 브랜드 컬러(일반적으로 청록색/시안과 다크 테마)를 주요 색상 체계로 사용
- 사용자 편의성: 자동 완성 기능이 있는 빠른 검색, 카테고리/주제별 필터, 명확한 대출 가능 여부 표시
- UI 단순성: 도서 목록을 위한 깔끔한 카드 기반 레이아웃, 도서 정보를 찾고 보는 데 필요한 최소한의 단계

**승인 시나리오**:

1. **주어진 상황** 학생이 모바일 앱을 열었을 때, **언제** 홈 화면을 볼 때, **그러면** 제목, 저자 및 대출 가능 여부가 표시된 모든 이용 가능한 도서 목록을 볼 수 있습니다
2. **주어진 상황** 학생이 도서를 탐색할 때, **언제** 도서 카드를 탭하면, **그러면** 설명, 출판 연도, 카테고리 및 현재 대출 가능 여부를 포함한 상세 정보를 볼 수 있습니다
3. **주어진 상황** 학생이 특정 도서를 찾고자 할 때, **언제** 검색창에 입력하면, **그러면** 일치하는 제목과 저자가 실시간으로 필터링되어 표시됩니다
4. **주어진 상황** 학생이 도서 상세 정보를 볼 때, **언제** 도서가 현재 대출 중이면, **그러면** 예상 반납일과 함께 "현재 대출 불가" 상태가 표시됩니다
5. **주어진 상황** 도서관에 여러 도서가 있을 때, **언제** 학생이 카테고리 필터를 적용하면, **그러면** 선택한 카테고리와 일치하는 도서만 표시됩니다

### 사용자 스토리 2 - 모바일 앱을 통한 도서 대출 요청 (우선순위: P2)

빌리고 싶은 도서를 찾은 학생들은 모바일 앱을 통해 대출 요청을 제출할 수 있습니다. 시스템은 인증을 위해 42 API와 연동되며 도서관 관리자가 처리할 수 있는 대출 요청을 추적합니다.

**이 우선순위인 이유**: 이것은 P1 탐색 기능을 확장하여 도서 대출이라는 핵심 도서관 기능을 가능하게 합니다. 탐색만으로도 유용하지만, 대출 요청은 완전한 도서관 경험을 제공하며 장기적 가치에 필수적입니다.

**독립적 테스트**: 42 API를 통해 인증하고, 이용 가능한 도서를 선택하고, 대출을 요청하며, 요청이 관리자 시스템에 나타나는지 확인함으로써 테스트할 수 있습니다. 대출 프로세스를 디지털화하여 가치를 제공합니다.

**승인 시나리오**:

1. **주어진 상황** 학생이 42 API 자격 증명으로 인증되었을 때, **언제** 이용 가능한 도서를 볼 때, **그러면** "대출 요청" 버튼이 표시됩니다
2. **주어진 상황** 학생이 "대출 요청"을 탭할 때, **언제** 요청을 확인하면, **그러면** 시스템은 학생 ID와 타임스탬프와 함께 대출 요청을 기록합니다
3. **주어진 상황** 도서에 활성 대출 요청이 있을 때, **언제** 다른 학생이 이를 볼 때, **그러면** 도서는 이용 가능 상태를 표시하고 추가 예약 요청을 허용하여 대기 중인 학생들의 FIFO 대기열을 생성합니다
4. **주어진 상황** 예약이 있는 동안 도서를 이용할 수 있게 되었을 때, **언제** 대기열의 첫 번째 사람에게 알림이 가면, **그러면** 예약이 자동으로 취소되고 다음 사람으로 넘어가기 전에 24시간 동안 대출을 완료할 수 있습니다
5. **주어진 상황** 학생이 대출을 요청했을 때, **언제** 프로필을 볼 때, **그러면** 활성 대출 요청 목록과 예약 대기열에서의 위치를 볼 수 있습니다
6. **주어진 상황** 대출 요청이 제출되었을 때, **언제** 관리자가 관리자 대시보드를 볼 때, **그러면** 요청이 대기 중인 대출 목록에 나타납니다

### 사용자 스토리 3 - 도서 추천 제출 (우선순위: P3)

학생들은 도서관 컬렉션에 추가되기를 원하는 도서를 추천할 수 있습니다. 이러한 추천은 분기별 또는 반기별로 수집되어 도서관 관리자가 검토합니다.

**이 우선순위인 이유**: 이것은 사용자 입력을 수집하여 도서관 큐레이션을 향상시키지만, 도서관은 이 기능 없이도 작동할 수 있습니다. 시간이 지남에 따라 컬렉션의 관련성을 향상시키는 좋은 기능입니다.

**독립적 테스트**: 앱을 통해 도서 추천을 제출하고 관리자 대시보드의 추천 컬렉션에 나타나는지 확인함으로써 테스트할 수 있습니다. 학생들에게 도서관 확장에 대한 목소리를 제공하여 가치를 제공합니다.

**승인 시나리오**:

1. **주어진 상황** 학생이 앱을 열었을 때, **언제** "도서 추천" 섹션으로 이동하면, **그러면** 도서 상세 정보(제목, 저자, 추천 이유)를 입력하는 양식이 표시됩니다
2. **주어진 상황** 학생이 추천 양식을 작성했을 때, **언제** 제출하면, **그러면** 시스템은 학생 ID와 제출 날짜와 함께 추천을 저장합니다
3. **주어진 상황** 추천 기간이 활성화되어 있을 때, **언제** 학생이 추천을 제출하면, **그러면** 다음 수집 주기에 검토될 것이라는 확인을 받습니다
4. **주어진 상황** 추천 기간이 종료되었을 때, **언제** 학생이 제출을 시도하면, **그러면** 다음 추천 기간이 열리는 시기를 나타내는 메시지가 표시됩니다
5. **주어진 상황** 여러 학생이 같은 도서를 추천했을 때, **언제** 관리자 대시보드를 볼 때, **그러면** 중복된 추천이 그룹화되어 총 요청 수가 표시됩니다

### 사용자 스토리 4 - 웹 대시보드를 통한 도서 카탈로그 관리 (우선순위: P1)

도서관 관리자는 웹 대시보드에 접속하여 도서 컬렉션을 관리합니다. 관련된 모든 상세 정보와 함께 새로운 도서를 카탈로그에 추가하고, 도서 정보를 업데이트하며, 더 이상 이용할 수 없는 도서를 제거하고, 전체 컬렉션을 검색/탐색할 수 있습니다.

**이 우선순위인 이유**: 관리자 기능은 도서관 유지에 필수적입니다 - 이것이 없으면 카탈로그를 업데이트할 수 없습니다. 이것은 사용자 탐색과 함께 P1으로, 두 기능 모두 작동하는 시스템에 필요합니다.

**독립적 테스트**: 웹 대시보드에 로그인하고, 도서를 추가/편집/제거하며, 변경 사항이 관리자 및 사용자 보기 모두에 반영되는지 확인함으로써 테스트할 수 있습니다. 도서관 유지보수를 가능하게 하여 가치를 제공합니다.

**승인 시나리오**:

1. **주어진 상황** 관리자가 웹 대시보드에 접속했을 때, **언제** "도서 추가"를 클릭하면, **그러면** 제목, 저자, ISBN, 카테고리, 설명, 출판 연도 및 수량에 대한 필드가 있는 양식이 표시됩니다
2. **주어진 상황** 관리자가 도서 양식을 작성했을 때, **언제** 제출하면, **그러면** 새 도서가 관리자 카탈로그와 모바일 앱 모두에 나타납니다
3. **주어진 상황** 관리자가 도서 목록을 볼 때, **언제** 카테고리별로 검색하거나 필터링하면, **그러면** 일치하는 도서를 표시하도록 결과가 업데이트됩니다
4. **주어진 상황** 관리자가 도서를 선택했을 때, **언제** "편집"을 클릭하면, **그러면** 도서 상세 정보를 수정하고 변경 사항을 저장할 수 있습니다
5. **주어진 상황** 관리자가 도서를 선택했을 때, **언제** "제거"를 클릭하면, **그러면** 확인 대화 상자가 표시되고 확인하면 도서가 시스템에서 제거됩니다
6. **주어진 상황** 도서에 활성 대출 또는 대출 요청이 있을 때, **언제** 관리자가 제거하려고 하면, **그러면** 시스템은 활성 대출에 대한 경고 메시지를 표시하고 강제 제거 옵션을 제공하며, 이는 모든 대기 중인 예약을 취소하고 도서를 영구적으로 이용 불가능으로 표시합니다

### 사용자 스토리 5 - 웹 대시보드를 통한 대출 추적 및 관리 (우선순위: P2)

도서관 관리자는 웹 대시보드를 통해 모든 대출 활동을 확인하고 관리합니다. 대기 중인 대출 요청을 확인하고, 요청을 승인하거나 거부하며, 현재 활성 대출을 확인하고, 도서 반납을 처리하며, 대출 이력을 확인할 수 있습니다.

**이 우선순위인 이유**: 이것은 모바일 대출 요청 기능(사용자 스토리 2)을 보완하여 대출 관리의 관리자 측을 제공합니다. 대출 도서관을 운영하는 데 필수적이지만 카탈로그 관리 기반에 의존합니다.

**독립적 테스트**: 대출 요청을 처리하고, 도서를 반납으로 표시하며, 대출 이력 보고서를 확인함으로써 테스트할 수 있습니다. 대출 추적을 디지털화하고 수작업 문서 작업을 줄여 가치를 제공합니다.

**승인 시나리오**:

1. **주어진 상황** 관리자가 대출 섹션에 접속했을 때, **언제** 대기 중인 요청을 볼 때, **그러면** 학생 이름, 도서 제목 및 요청 날짜가 있는 목록이 표시됩니다
2. **주어진 상황** 관리자가 대출 요청을 볼 때, **언제** "승인"을 클릭하면, **그러면** 대출 상태가 "활성"으로 변경되고 도서 이용 가능성이 업데이트됩니다
3. **주어진 상황** 활성 대출이 존재할 때, **언제** 관리자가 "반납으로 표시"를 클릭하면, **그러면** 대출 상태가 "완료"로 변경되고 도서가 다시 이용 가능해집니다
4. **주어진 상황** 관리자가 연체 항목을 추적하고자 할 때, **언제** 활성 대출을 볼 때, **그러면** 반납 날짜가 지난 대출이 강조 표시되거나 플래그가 지정됩니다
5. **주어진 상황** 대출 데이터가 존재할 때, **언제** 관리자가 대출 이력 보고서를 요청하면, **그러면** 날짜 범위, 학생 또는 도서별로 필터링된 데이터를 확인/내보낼 수 있습니다

### 사용자 스토리 6 - 웹 대시보드를 통한 도서 추천 검토 (우선순위: P3)

도서관 관리자는 웹 대시보드를 통해 수집된 도서 추천에 접근합니다. 현재 수집 기간 동안 제출된 모든 추천을 확인하고, 각 도서를 요청한 학생 수를 확인하며, 추천을 승인, 거부 또는 검토 중으로 표시할 수 있습니다.

**이 우선순위인 이유**: 이것은 사용자 스토리 3의 관리자 대응 부분입니다. 데이터 기반 컬렉션 확장을 가능하게 하지만 핵심 도서관 운영에 필수적이지는 않습니다.

**독립적 테스트**: 추천 대시보드를 확인하고, 날짜 범위별로 필터링하며, 추천 상태를 업데이트함으로써 테스트할 수 있습니다. 컬렉션 검토 프로세스를 간소화하여 가치를 제공합니다.

**승인 시나리오**:

1. **주어진 상황** 관리자가 추천 섹션에 접속했을 때, **언제** 목록을 볼 때, **그러면** 도서 상세 정보, 요청자 수 및 제출 날짜가 있는 모든 제출된 추천이 표시됩니다
2. **주어진 상황** 여러 학생이 같은 도서를 추천했을 때, **언제** 목록을 볼 때, **그러면** 추천이 그룹화되어 총 요청 수가 표시됩니다
3. **주어진 상황** 관리자가 추천을 검토할 때, **언제** "승인됨"으로 표시하면, **그러면** 선택적으로 카탈로그에 직접 추가하거나 향후 구매를 위해 추적할 수 있습니다
4. **주어진 상황** 새로운 추천 기간이 시작될 때, **언제** 대시보드를 볼 때, **그러면** 이전 기간 추천이 보관되고 목록에는 현재 기간만 표시됩니다
5. **주어진 상황** 관리자가 경향을 분석하고자 할 때, **언제** 추천 통계를 볼 때, **그러면** 가장 많이 요청된 카테고리 또는 저자를 볼 수 있습니다

- **Active Loan(활성 대출)**: 학생 ID, 도서 참조, 대출 날짜, 예상 반납일 및 완료 시 실제 반납일을 포함한 진행 중인 도서 대출을 나타냅니다
- **Book Suggestion(도서 추천)**: 제안된 도서 상세 정보(제목, 저자), 학생 ID, 추천 이유, 제출 날짜 및 수집 기간을 포함한 도서관 추가에 대한 학생의 제안을 나타냅니다

- **Collection Period(수집 기간)**: 도서 추천이 수락되는 시간 창(분기별 또는 반기별)

- **SC-010**: 시스템은 성능 저하 없이 50명의 동시 모바일 앱 사용자를 처리합니다
- **SC-011**: 모바일 앱과 웹 대시보드 간의 데이터 동기화는 변경 후 10초 이내에 발생합니다
- **SC-012**: 모바일 앱은 오프라인일 때 캐시된 데이터로 작동하며 연결이 복원되면 변경 사항을 동기화합니다

**기능 채택 지표**

- **SC-013**: 42 학습 공간 학생의 70%가 출시 첫 달 내에 모바일 앱을 설치하고 사용합니다
- **SC-014**: 주당 평균 20-30건의 대출 요청이 처리됩니다(활발한 사용을 나타냄)
- **SC-015**: 학생 도서 추천이 수집 기간당 50-100건의 제출에 도달합니다
- **SC-016**: 대출 요청의 95%가 관리자에 의해 24시간 이내에 처리됩니다

## 가정

- 학생들은 iOS 또는 Android 운영 체제를 갖춘 스마트폰을 가지고 있습니다
- 42 API는 안정적인 인증 엔드포인트와 학생 식별 데이터를 제공합니다
- 모바일 앱 사용을 위해 42 학습 공간에서 인터넷 연결을 사용할 수 있습니다
- 단일 관리자 또는 소규모 관리 팀이 시스템을 관리합니다
- 도서는 학습 공간의 서가에 있는 물리적 항목이며, 디지털/전자책이 아닙니다
- 대출 기간은 관리자가 정의합니다(기본 가정: 2주 대출)
- 추천을 위한 수집 기간은 연 2-4회(분기별 또는 반기별) 발생합니다
- 도서 카탈로그는 현재 500권에서 최대 1000권까지 점진적으로 증가할 것입니다
- 학생들은 42 커뮤니티 내에서 신뢰할 수 있는 사용자입니다(초기에는 복잡한 사기 방지가 필요하지 않음)
- 웹 대시보드 관리자는 데스크톱/노트북 컴퓨터에서 시스템에 접근합니다
- 기본 도서 메타데이터(제목, 저자, 카테고리)는 관리자가 수동으로 입력합니다
- ISBN은 일부 교육 자료에 ISBN 번호가 없을 수 있으므로 선택 사항입니다

## 종속성

- **42 API 접근**: 인증 및 학생 식별은 안정적인 42 API 엔드포인트에 의존합니다
- **호스팅 인프라**: 시스템은 백엔드 서비스를 위한 클라우드 호스팅 또는 서버 인프라가 필요합니다(이용 가능하다고 가정)
- **초기 도서 데이터**: 도서관 카탈로그는 사용자 출시 전에 초기 500권의 도서로 채워져야 합니다
- **관리자 교육**: 관리 사용자는 웹 대시보드 기능에 대한 기본 교육이 필요합니다
- **네트워크 연결**: 모바일 앱과 웹 대시보드 모두 실시간 동기화를 위해 인터넷 연결이 필요합니다

## 향후 확장성

이 명세서는 다음 단계로 계획된 확장과 함께 핵심 도서관 관리 기능에 중점을 둡니다:

- **2단계**: 자동화된 알림 및 연체 알림을 포함한 전체 대출 수명 주기 관리(참고: 대출 연장 기능은 계획되지 않음)
- **3단계**: 빠른 도서 조회 및 대출을 위한 QR 코드 또는 바코드 스캔
- **4단계**: 사용자 행동을 기반으로 한 독서 통계 및 추천 엔진
- **5단계**: 자동화된 메타데이터 채우기를 위한 외부 도서 데이터베이스(Google Books, Open Library)와의 연동
- **6단계**: 컬렉션의 도서에 대한 토론 포럼 또는 리뷰 시스템
- **7단계**: 예약 패턴 및 도서 인기 추세에 대한 고급 분석
